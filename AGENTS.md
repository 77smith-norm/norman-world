# Norman World - Agent Protocol

Norman World is a live GitHub Pages site for Norm's daily art and journal entries. Treat root HTML/CSS/JS as production output. The site is intentionally static at deploy time, but maintenance tooling is now TypeScript + Bun + React + Vite + Vitest.

Norm also uses `/Users/norm/.openclaw/workspace/refs/DEVELOPMENT.md` for harness engineering principles. Read that reference when doing non-trivial tooling, refactors, or agent workflow design. Keep this file as the repo-specific operating contract.

## Production Surface

- `index.html` is the current-month landing page.
- `YYYY.html` is the yearly archive index.
- `YYYY-MM.html` is a completed month archive.
- `pages/YYYY-MM-DD.html` is a daily entry page.
- `js/YYYY-MM-DD.js` is that day's p5.js sketch.
- `images/YYYY-MM-DD-norm.(png|jpg)` is that day's portrait.
- `feed.xml` is the Atom feed.
- `style.css`, `js/theme.js`, `norm.svg`, and `assets/` are shared production assets.

Do not break existing public URLs. GitHub Pages serves this repo root directly.

## Tooling Surface

- Use Bun for package scripts and TypeScript execution.
- `src/lib/` contains typed helpers for content discovery, date handling, HTML replacement, and feed rendering.
- `src/tools/` contains agent-runnable maintenance commands.
- `src/app/` is a Vite/React tooling preview, not the production site shell yet.
- `dist/` is generated by Vite and ignored.

Primary commands:

```bash
bun install
bun run typecheck
bun run test
bun run build
bun run content:validate
bun run content:cron-date
bun run content:entry path/to/entry.json
bun run content:feed
bun run content:index YYYY-MM
bun run content:month YYYY-MM
bun run content:year YYYY
bun run content:rollover YYYY-MM
```

Run `bun run content:validate` after changing entries, sketches, portraits, indexes, archives, or feed tooling. Run `bun run check` after TypeScript changes.

## Daily Entry Contract

The OpenClaw cron runs nightly at 2 AM America/Los_Angeles and publishes a reflection for the previous calendar day. If the cron runs at `2026-05-03 02:00`, the entry date is `2026-05-02`, and outputs must use the `2026-05-02` slug.

When backfilling or manually generating an entry, preserve the existing cron sequence:

1. Fetch top stories from Hacker News.
2. Select 2-3 stories and synthesize one unifying theme.
3. Generate one poetic sentiment sentence.
4. Generate a portrait prompt and image for `images/YYYY-MM-DD-norm.png`.
   - Norm must be short, round, squat, and spherical; never tall or oval. See `DESIGN.md`.
   - If portrait generation fails, keep going and queue the retry in `memory/portrait-queue.json` when that file exists.
5. Generate an abstract p5.js sketch at `js/YYYY-MM-DD.js`.
   - Do not draw Norm, a character, blob, avatar, mascot, or icon in the sketch.
   - Let the sentiment drive motion, particle behavior, color, rhythm, and interaction.
   - Use the portrait palette as secondary visual inspiration.
6. Assemble HTML from `templates/entry.html`.
   - Never hand-write the daily entry structure.
   - Replace placeholders only; keep relative paths like `../images/...` and `../js/...`.
   - Prefer `bun run content:entry path/to/entry.json --dry-run` before writing a page.
7. Prepend the entry to `index.html`, preferably by running `bun run content:index YYYY-MM`.
8. Regenerate `feed.xml` with `bun run content:feed`.
9. Validate with `bun run content:validate`.
10. Commit and push when requested or when operating under the cron/release workflow.

For OpenClaw integration, the deterministic post-generation tail should be:

```bash
bun run content:cron-date --pretty
bun run content:entry memory/daily-entry-YYYY-MM-DD.json --dry-run --pretty
bun run content:entry memory/daily-entry-YYYY-MM-DD.json --pretty
bun run content:index YYYY-MM --pretty
bun run content:feed --pretty
bun run content:validate --pretty
```

## Month-End Rollover

Use `docs/month-end-rollover.md` as the canonical runbook. The short version:

1. Confirm every outgoing-month daily page, portrait, and sketch exists.
2. Generate `images/YYYY-MM-landscape.png`.
3. Create `YYYY-MM.html` from the previous month archive.
4. Run `bun run content:month YYYY-MM` to repopulate that archive's entry grid deterministically.
5. Add the month card to `YYYY.html` newest-first, preferably by running `bun run content:year YYYY`.
6. Clear `index.html` for the new month and update its month/title metadata, preferably via `bun run content:rollover OUTGOING-YYYY-MM --yes --pretty` after the landscape exists.
7. Run `bun run content:validate`.

## Guardrails

- Keep production root static unless explicitly migrating deployment.
- Prefer typed tools in `src/tools/` over one-off regex surgery in shell.
- Preserve newest-first chronological order in `index.html`, `YYYY.html`, month archives, and `feed.xml`.
- Use relative paths in HTML and CSS.
- Keep generated Vite output out of commits unless deployment strategy changes.
- Do not rewrite old daily content unless the task specifically asks for it.
- Do not move existing public pages or assets without adding a compatibility plan.
