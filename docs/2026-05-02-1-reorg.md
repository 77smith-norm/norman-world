## Active Constraints

- Do not move existing public pages or assets.
- Do not migrate GitHub Pages deployment away from repo-root static output in this pass.
- Do not rewrite historical daily content unless a tool-generated output requires a tiny normalization and is verified.
- Prefer typed tools over ad hoc shell/regex surgery.
- Every content tool should move toward JSON output, semantic exit codes, and `--dry-run` where writes occur.
- Daily entries must still be assembled from `templates/entry.html`.
- The OpenClaw nightly cron runs at 2 AM America/Los_Angeles for the previous calendar day. A run on `YYYY-MM-DD 02:00` publishes `YYYY-MM-(DD-1)` and must use that previous-day slug everywhere.
